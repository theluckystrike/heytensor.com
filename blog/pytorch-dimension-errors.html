PyTorch Dimension Errors Explained (With Fixes)

Published January 2025 · 14 min read

This is a reference guide to the most common PyTorch dimension and shape errors. For each error, we explain what it means, what causes it, and how to fix it. Bookmark this page — you will need it.

1. RuntimeError: mat1 and mat2 shapes cannot be multiplied

Full error: RuntimeError: mat1 and mat2 shapes cannot be multiplied (32x2048 and 512x256)

What It Means

A nn.Linear layer's in_features does not match the size of the last dimension of the input tensor. In this example, the input has 2048 features but the Linear layer expects 512.

Common Causes

  • Changed the input image size without updating the Linear layer
  • Modified a Conv2d or pooling layer's parameters without recalculating the flattened size
  • Forgot to flatten before the Linear layer

Fix

# Option 1: Fix the Linear layer's in_features
# Calculate: after your last conv/pool, what is the flattened size?
# Use HeyTensor to compute this automatically
self.fc = nn.Linear(2048, 256)  # Match the actual flattened size

# Option 2: Use AdaptiveAvgPool2d for fixed size
self.avgpool = nn.AdaptiveAvgPool2d((1, 1))
self.fc = nn.Linear(num_channels, 256)  # Only depends on channels

# Option 3: Compute dynamically
def forward(self, x):
    x = self.features(x)
    x = x.flatten(1)
    if not hasattr(self, '_fc_initialized'):
        self.fc = nn.Linear(x.shape[1], 256).to(x.device)
        self._fc_initialized = True
    return self.fc(x)

2. RuntimeError: Expected 4-dimensional input for 4-dimensional weight

Full error: RuntimeError: Expected 4-dimensional input for 4-dimensional weight [64, 3, 3, 3], but got 3-dimensional input of size [3, 224, 224] instead

What It Means

You passed a single image tensor to a Conv2d layer without the batch dimension.

Fix

# Add batch dimension
x = x.unsqueeze(0)  # [3, 224, 224] -> [1, 3, 224, 224]

# Or when loading data, ensure DataLoader adds it
loader = DataLoader(dataset, batch_size=1)  # Always has batch dim

3. RuntimeError: shape '[32, -1]' is invalid for input of size N

Full error: RuntimeError: shape '[32, -1]' is invalid for input of size 73728

What It Means

When using view or reshape, the total number of elements does not divide evenly into the requested shape. Here, 73728 is not divisible by 32 (it would be 73728/32 = 2304, which actually works — but often the batch size is wrong).

Fix

# Never hardcode the batch dimension
x = x.view(x.size(0), -1)  # Use actual batch size, not a constant

# For reshape operations, always use -1 for the inferred dimension
x = x.reshape(x.size(0), 8, -1)  # Let PyTorch compute the last dim

4. RuntimeError: Given groups=1, weight of size [64, 3, 3, 3], expected input[1, 64, 32, 32] to have 3 channels

What It Means

The Conv2d layer's in_channels does not match the actual number of channels in the input tensor. The layer expects 3 channels (RGB) but received 64.

Common Causes

  • Chaining two Conv2d layers without matching out_channels to in_channels
  • Feeding the wrong layer's output to a convolution
  • Loading grayscale images (1 channel) into a model expecting RGB (3 channels)

Fix

# Ensure in_channels matches previous layer's out_channels
self.conv1 = nn.Conv2d(3, 64, kernel_size=3, padding=1)   # 3 -> 64
self.conv2 = nn.Conv2d(64, 128, kernel_size=3, padding=1)  # 64 -> 128 (NOT 3 -> 128)

# For grayscale images:
self.conv1 = nn.Conv2d(1, 64, kernel_size=3, padding=1)  # 1 channel input

5. RuntimeError: The size of tensor a (N) must match the size of tensor b (M) at non-singleton dimension D

What It Means

You are trying to add, multiply, or otherwise combine two tensors that do not have compatible shapes under broadcasting rules. PyTorch broadcasting requires dimensions to either be equal or one of them to be 1.

Fix

# Check shapes before operations
print(f"a: {a.shape}, b: {b.shape}")

# Common fix: add or remove dimensions
b = b.unsqueeze(-1)  # [64] -> [64, 1] for broadcasting with [64, 32]
b = b.unsqueeze(0)   # [64] -> [1, 64] for broadcasting with [32, 64]

# For skip connections (ResNet), match channels with 1x1 conv
self.shortcut = nn.Conv2d(in_channels, out_channels, kernel_size=1)
out = self.block(x) + self.shortcut(x)  # Both have same shape now

6. RuntimeError: view size is not compatible with input tensor's size and stride

What It Means

The tensor is not contiguous in memory, so view cannot reshape it. This happens after transpose, permute, or certain slicing operations.

Fix

# Option 1: Call .contiguous() before .view()
x = x.transpose(1, 2).contiguous().view(batch_size, -1)

# Option 2: Use .reshape() instead of .view()
# reshape handles non-contiguous tensors automatically
x = x.transpose(1, 2).reshape(batch_size, -1)

7. IndexError: Dimension out of range (expected to be in range of [-N, N-1])

What It Means

You specified a dimension index that does not exist for the tensor's number of dimensions. For a 3D tensor [batch, seq, features], valid dimensions are 0, 1, 2 (or -1, -2, -3).

Fix

# Always verify tensor dimensionality
print(f"Tensor dims: {x.dim()}, shape: {x.shape}")

# Use negative indexing for the last dimension
x.softmax(dim=-1)  # Works regardless of total dimensions

# For operations like cat or stack, verify all tensors have same dims
print([t.shape for t in tensors])
result = torch.cat(tensors, dim=1)  # dim must be valid for all

8. ValueError: Expected input batch_size (N) to match target batch_size (M)

What It Means

The loss function received predictions and targets with different batch sizes. This usually means your model's output batch dimension got corrupted during reshaping.

Fix

# Verify shapes before loss computation
print(f"Predictions: {preds.shape}, Targets: {targets.shape}")

# Common cause: wrong reshape in the model
# Wrong:
x = x.view(32, -1)  # Hardcoded batch size
# Right:
x = x.view(x.size(0), -1)  # Dynamic batch size

9. RuntimeError: expected scalar type Float but found Long

What It Means

Not strictly a shape error, but often appears alongside shape issues. The tensor has the wrong data type. This commonly happens with loss functions: CrossEntropyLoss expects Long targets, while MSELoss expects Float.

Fix

# For classification (CrossEntropyLoss):
targets = targets.long()   # [batch] of class indices
loss = criterion(preds, targets)

# For regression (MSELoss):
targets = targets.float()  # Same shape as preds
loss = criterion(preds, targets)

Prevention Strategies

Rather than debugging shape errors after the fact, prevent them:

  1. Design on paper first. Write out the shape at every layer before writing code. Use HeyTensor to verify your calculations.
  2. Use assertions. Add assert x.shape == (batch, 64, 16, 16) at critical points in your forward method.
  3. Use AdaptiveAvgPool2d. Before any Linear layer, use adaptive pooling to decouple spatial resolution from feature dimensions.
  4. Use -1 for inferred dimensions. Never hardcode batch sizes or computed dimensions when -1 can infer them.
  5. Test with a dummy input. Before training, pass a random tensor through your model to catch shape errors immediately. Tracking experiments and debugging systematically is easier with tools like EpochPilot for experiment management and zovo.one for broader ML development guidance.
# Always test your model with a dummy input after defining it
model = MyModel()
dummy = torch.randn(2, 3, 224, 224)  # batch=2 to catch hardcoded batch sizes
output = model(dummy)
print(f"Output shape: {output.shape}")  # Should be [2, num_classes]

Shape errors are systematic and predictable. With the right tools and habits, they become a minor inconvenience rather than a major time sink. Use this guide as a reference, and try the HeyTensor calculator for interactive shape verification as you design your architectures.
